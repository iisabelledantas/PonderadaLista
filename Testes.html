Testes Unitários

Os testes unitários têm como objetivo garantir a corretude das regras de negócio da aplicação de forma isolada, validando o comportamento das principais camadas sem dependência de serviços externos.

Para garantir isolamento total, são utilizados mocks para o banco de dados e serviços da AWS, permitindo testes rápidos, determinísticos e independentes do ambiente externo.

Ferramentas​

FerramentaVersãoFinalidade
JUnit 5-Framework de testes unitários
Mockito-Criação de mocks, stubs e verificação de interações
mockito-kotlin5.2.1Suporte à sintaxe Kotlin no Mockito (any(), verify, etc.)
MockitoExtension-Integração JUnit 5 + Mockito via @ExtendWith

Cobertura dos Testes​

TelemetriaConsumerTest (TelemetriaProcessor)​

Testa a lógica de processamento do lote de mensagens SQS e persistência em batch no banco.

TesteCenárioResultado Esperado
deve processar batch com mensagem valida sem lancar excecaoUma mensagem JSON válidaNenhuma exceção lançada
deve executar batch insert ao processar mensagem validaUma mensagem válidaaddBatch() chamado 1x, executeBatch() chamado 1x
deve executar batch insert para cada mensagem valida no loteTrês mensagens válidasaddBatch() chamado 3x, executeBatch() chamado 1x
deve descartar mensagem com body invalido sem lancar excecaoBody vazioNenhuma exceção, addBatch() nunca chamado
deve processar mensagens validas e descartar invalidas no mesmo batch2 válidas + 1 inválidaaddBatch() chamado 2x, batch executado normalmente
deve reutilizar conexao existente em warm startDuas invocações consecutivasprepareStatement chamado 2x, close() nunca chamado
deve lancar excecao quando executeBatch falharexecuteBatch() lança RuntimeExceptionExceção propagada, autoCommit = false verificado
nao deve fazer nada com lista vaziaLista de mensagens vaziaNenhuma interação com a conexão

TelemetriaControllerTest​

Testa o endpoint REST e a delegação ao SqsService.

TesteCenárioResultado Esperado
deve retornar 200 OK ao receber dados válidosDTO válido recebido via POSTStatus HTTP 200
deve chamar o sqsService ao receber dados válidosDTO válido recebidosqsService.sendMessage() chamado exatamente 1x
deve lancar excecao ao falhar serializaçãoObjectMapper lança exceçãoExceção propagada ao controller

SqsServiceTest​

Testa o envio de mensagens para a fila SQS via AWS SDK.

TesteCenárioResultado Esperado

deve lancar excecao quando o sqsClient falharSqsClient lança RuntimeExceptionExceção propagada ao chamador
